> **FzBoard** is a web-based data analysis dashboard built with **Django 6.0** and **Tailwind CSS 3.4**, styled with the **Catppuccin Mocha** color palette and designed for Docker deployment.
>
> ⚠️ *The dashboard interface is currently a **mockup** — most data shown on the dashboard page is hardcoded.*

## Infrastructure & Deployment

### Docker

The application is containerized using a **multi-stage Dockerfile** based on `python:3.12-slim`:

1. **System dependencies** — Installs `curl`, `build-essential`, `libpq-dev`, and **Node.js 20 LTS** (via NodeSource).
2. **Python dependencies** — Copies `requirements.txt` and runs `pip install`.
3. **Node dependencies** — Copies `package.json` and runs `npm install`.
4. **Tailwind build** — Runs `npm run build` to compile CSS.
5. **Entrypoint** — Runs `start.sh` which launches Tailwind in watch mode and the Django dev server.

**Exposed port:** `8000`

### Docker Compose (`docker-compose.yml`)

```yaml
services:
  web:
    build: .
    command: bash ./start.sh
    volumes:
      - .:/app                  # Bind mount for live code changes
      - /app/node_modules       # Prevent overwriting node_modules
    ports:
      - "8000:8000"
    environment:
      - DEBUG=1
```

### Startup Script (`start.sh`)

```bash
#!/bin/bash
npm run watch &               # Tailwind CSS watcher (background)
python manage.py runserver 0.0.0.0:8000  # Django dev server (foreground)
```

This allows **hot reloading** of both CSS (Tailwind) and Python (Django) during development.

## Application: `dashboard`

The `dashboard` is the single Django app that contains all application logic.

### Models

**`dashboard/models.py`**

```python
class User(AbstractUser):
    pass
```

- Extends Django's `AbstractUser` with **no additional fields** (yet).
- Registered as the project's `AUTH_USER_MODEL`.
- Registered in the Django admin via `admin.site.register([User])`.
- This pattern allows for easy future extension (e.g., adding profile fields, roles, etc.).

Extends Django's `UserCreationForm` with the following fields:

| `email`      | `EmailInput`    | Required                             |

| `last_name`  | `TextInput`     | Required, max 30 chars               |

All widgets use the CSS class `form-input-hover` for consistent styling.

#### `LoginForm`

A simple `forms.Form` with:

| Field      | Widget          |
|------------|-----------------|
| `username` | `TextInput`     |
| `password` | `PasswordInput` |

### Views & URL Routing

**`dashboard/urls.py`**

| URL Pattern | View Function        | Name         | Auth Required |
|-------------|----------------------|--------------|---------------|
| `/`         | `dashboard()`        | `dashboard`  | ✅ Yes         |
| `/login/`   | `login_view()`       | `login`      | ❌ No          |
| `/register/`| `register_view()`    | `register`   | ❌ No          |
| `/logout/`  | `logout_view()`      | `logout`     | ❌ No          |

**`dashboard/views.py`** — View functions:

| View             | Method(s)    | Behavior                                                                              |
|------------------|--------------|---------------------------------------------------------------------------------------|
| `dashboard()`    | GET          | Protected by `@login_required`. Renders the dashboard mockup.                         |
| `register_view()`| GET / POST   | Shows registration form. On valid POST: creates user, auto-logs in, redirects to `/`. Redirects authenticated users to `/`. |
| `login_view()`   | GET / POST   | Shows login form. On valid POST: authenticates user, redirects to `/`. Redirects authenticated users to `/`. |
| `logout_view()`  | Any          | Logs out the user and redirects to `/login/`.                                         |

> **Localization note:** All user-facing messages (flash messages) are in **Spanish** (e.g., *"¡Cuenta creada exitosamente!"*, *"Bienvenido de vuelta"*).

- 6-field registration form with responsive 2-column grid for name fields.

#### `dashboard.html`

  - Logo link.
  - Desktop navigation tabs (Dashboard, Proyectos, Formatos).
  - User info display (`username`, `email`).
  - User avatar icon.
  - Logout button (POST form with CSRF token).

  - **Latencia Media** — 0.45s, Rust Backend Active (yellow accent).
  - **Tokens** — 8.4M, healthy status (red accent).
- **Recent files table** — 3 hardcoded rows with status badges (Completado, Procesando, Error).
- **Drag & drop area** — File upload zone (PDF, CSV, JSON, TXT, max 500MB).

> ⚠️ All dashboard data is **hardcoded mockup data** — no backend data fetching is implemented yet.

## Authentication Flow

```
┌────────────────┐     ┌─────────────────┐     ┌──────────────────┐
│  /login/       │────▶│  Authenticate   │────▶│  / (Dashboard)   │
│  (LoginForm)   │     │  (Django Auth)  │     │  @login_required │
└────────────────┘     └─────────────────┘     └──────────────────┘
        ▲                                              │
        │                                              │
        │  redirect                        logout (POST)
        │                                              │
        ▼                                              ▼
┌────────────────┐                             ┌──────────────────┐
│  /register/    │                             │  /logout/        │
│  (UserCreation │────▶ auto-login ───────────▶│  redirect →      │
│   Form)        │                             │  /login/         │
└────────────────┘                             └──────────────────┘
```

1. **Unauthenticated users** accessing `/` are redirected to `/login/` (via `LOGIN_URL` setting).
2. **Registration** creates a user, auto-logs them in, and redirects to the dashboard.
3. **Login** authenticates against the `User` model and redirects to `/`.
4. **Logout** terminates the session and redirects to `/login/`.
5. **Already authenticated users** visiting `/login/` or `/register/` are redirected to `/`.
6. **Sessions expire** after 1 hour or when the browser is closed.
7. **Password validation** is enforced via Django's built-in validators: `UserAttributeSimilarityValidator`, `MinimumLengthValidator`, `CommonPasswordValidator`, `NumericPasswordValidator`.
